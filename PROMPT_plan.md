ULTIMATE GOAL: We want to achieve a fully functional Mountaineer V2 — a macOS SMB share failover manager that keeps network shares usable across Thunderbolt/fallback interface changes. The architecture is single-mount only (one interface mounts a share at a time, eliminating CORE vs CORE-1 path collisions). Phase 1 delivers a feature-complete Rust CLI with: automatic failover (TB→fallback), user-controlled recovery (fallback→TB with open-file safety), stable symlinks (`~/Shares/<SHARE> → /Volumes/<SHARE>`), favorites management, subfolder aliases, bulk mount/unmount, background monitoring via reconciliation loop, launchd integration, and `--json` output for scripting. Phase 2 delivers a native Swift (NOT GPUI) menu bar UI that calls the same engine functions as CLI — covering one-click TB recovery, favorites/alias management, bulk operations, live per-share status, and quick-access actions. Config lives at `~/.mountaineer/config.toml`, state at `~/.mountaineer/state.json`, logs at `~/Library/Logs/mountaineer.log`. Consider missing elements and plan accordingly.
